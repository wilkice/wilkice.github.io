外网连接到内部服务器的服务的几种方式

从外部连接到其他局域网的内部服务器的 SSH, web 服务等，可采用 SSH ProxyCommand、SSH 端口转发、iptables 设置等

示例

服务器 A: root@106.75.189.222，内部地址：172.16.0.162

服务器 B: 内部地址：root@172.16.0.98

外部服务器 C 即我们的电脑💻，103.45.22.31

服务器 A 可被外部直接访问，服务器 B 不可被外部直接访问。外部访问服务器 B 必须经过服务器 A 的帮助才可以，而外部连接服务器 A 时，通过网络传过来的参数只有3个 — 源地址，目的地址，端口。因此，要想实现外部能访问服务器 B，则要想办法让传过来的这3个参数能被解析到服务器 B。

方法1.1：利用 SSH 的命令代理打开终端窗口

连接过程：服务器 C 连接到服务器 A 打开一个隧道，不打开终端，服务器 C 通过这个隧道连接服务器 B。因此外部服务器 C 需要具有 A, B 上用户的凭据（密码或密钥），A 上不需要有 B 的密码或密钥

在外部服务器 C 上执行

⚠️：输入这条命令会输入2次密码，第一次是服务器 A 的密码，第二次是服务器 B 的密码。如果要免密登录的话，需要把外部服务器 C 的用户公钥利用 ssh-copy-id 传给服务器 A 和 B 的相应用户，服务器 A 上不需要服务器 B 上用户的密钥。

方法1.2：利用 SSH 的命令代理打开终端窗口

连接过程：服务器 C 登录到 服务器 A 并打开一个终端，在这个终端里输入 ssh root@172.16.0.98 连接到 服务器 C。因此此过程只需要 C 知道 A 的凭据，A 知道 B 的凭据。和第一种方法注意区分。
