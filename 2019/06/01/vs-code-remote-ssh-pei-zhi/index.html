使用 GCP 创建虚拟主机，通过 Vs code 远程开发

GCP + Vs code remote ssh

Follow these steps to set remote development environment on GCP.

GCP settings

  1. Create computer instance follow this tutorial by CS231N

  2. SSH to instance with browser

  3. Change the following lines so we can connect to this instance by ssh key not password. If you like, you can use password authentication. Maybe need a restart for ssh /etc/init.d/ssh restart

  4. Add SSH key to GCP. Following my reference at the bottom of this page.

  5. Connect to instance in terminal

  6. Add user to sudo group

Vs code settings

  1. Download Vs Code Insider.

  2. Install Remote Development Extension Pack

  3. Follow this docs. It’s detailed with Videos and Images.

    If you meet some problems when using the internal terminal in vs code, please refer to this issue

  4. You can follow this Youtube Video too.

FAQ:

  1. Can I use us region computer instance if I am in China?

    Sure you can. Maybe you need to set all proxy to https://127.0.0.1:1080.

  2. Do I need to install miniconda?

    No, miniconda is preinstalled.

  3. How much does it cost?

    n1-highmem-2 (2 vCPUs, 13 GB memory) instance in us: $ 0.1184/h

    K80 GPU: about $ 0.45/h

    Disk and Internet traffic don’t cost much.
