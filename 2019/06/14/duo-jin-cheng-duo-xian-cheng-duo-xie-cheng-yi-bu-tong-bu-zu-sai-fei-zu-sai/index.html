多进程，多线程，多协程，异步，同步，阻塞，非阻塞

发表于 2019-06-14 阅读次数：

进程，线程，协程，同步，异步，阻塞，非阻塞的区别😊

进程：由 CPU 调度，适合 CPU 密集型操作，如计算
一个运行的程序就是进程。进程是资源分配的最小单位，有自己的独立内存空间。进程间资源不共享，开销大。
多进程：存在于多核 CPU

线程：由 CPU 调度，适合 IO 密集型操作，如爬虫，数据读写
线程：执行的基本单位，依赖于进程，一个进程至少有一个线程。多个线程共享内存（数据及全局变量）。
问题：资源竞争
解决：GIL 锁（Cpython）

多协程：由用户调度
协程：协程拥有自己的寄存器，上下文和栈。协程调度时，将寄存器，上下文和栈保存到其他地方，工作的时候再回复之前的状态。开销小，可以不加锁访问全局变量，切换快。
优势：

  • 效率高
  • 不需要锁。

多核一般是多进程 + 多协程

并发：多个任务在一个时间间隔内同时运行，每一时刻只有一个任务在运行 Cpython 中多线程是并发

并行：同一时刻多个任务同时在运行
多核 CPU 的多进程是并行，单核 CPU 的多进程也是并发

异步：
不同程序为了完成某一任务，无需相互通信。无序

同步：
不同程序为了完成某一任务，靠某种通信方式以协调一致。有序

阻塞：

程序发出请求后，必须接收到返回才能继续操作，处于挂起的状态。包括网络 I/O 阻塞，磁盘 I/O 阻塞，输入阻塞

非阻塞：

程序发出请求后，不用等待返回值就可继续其他操作。程序下必须要有独立的子单元。
